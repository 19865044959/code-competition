## 整体通信说明

每局比赛双方选手代码同步启动，选手程序以 HTTP Server 方式运行并监听指定端口；判题系统通过 HTTP 请求下发当前全量游戏状态，接收选手返回的全体英雄调度指令。

**核心设计**：判题器每回合将**全部信息**（地图、己方英雄/建筑/仓库/任务、敌方可见信息、世界新闻、当前激活任务等）一次性返回给选手；选手在一个 HTTP 响应中返回**3 名英雄的本回合动作指令**。不存在"3 个英雄 3 个 API"的设计——全部信息由判题器统一管理和下发。

程序启动命令示例：

```bash
bash run.sh port
```

- **port**：系统传入的监听端口号
- **服务监听地址**：固定为 `0.0.0.0`

# 1 请求报文（Request）顶层结构

判题器 → 选手。每回合发送一次。

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| roundNo | Int | 当前回合数（1-2000） |
| dayNo | Int | 当前天数（1-10） |
| phase | String | 当前阶段：`day` = 白天（120回合），`night` = 黑夜（80回合） |
| roundInPhase | Int | 当前阶段内第几回合（从 1 开始） |
| map | Map | 地图信息（地形+资源+野兽） |
| team | Team | 己方队伍全部信息（英雄/建筑/仓库/任务记录） |
| enemy | Enemy | 敌方可见信息（英雄/建筑/金币/积分） |
| worldNews | WorldNews | 当日晨间新闻（仅在每天第 1 回合推送；其余回合为 null） |
| nightWarning | NightWarning | 黄昏预警（仅在白天最后一回合推送，告知今晚兽潮种类与数量；其余回合为 null） |
| taskOfficer | TaskOfficer | 任务官信息（仅当开拓者位于任务官坐标(13,10)时返回；其余回合为 null） |
| activeTask | ActiveTask | 当前激活的自进化类任务信息（无激活任务时为 null） |
| tradeRequest | TradeRequest | 对方发起的交易请求（仅当对方开拓者向己方发起交易时返回；其余回合为 null） |
| lastRound | LastRound | 上回合各英雄动作执行结果 |
| errors | Error[] | 本轮产生的错误信息数组 |

## 1.1 Map — 地图信息

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| width | Int | 地图宽度（25） |
| height | Int | 地图高度（25） |
| terrains | Terrain[] | 地形数组（仅包含非平地区域，减少数据量） |
| resources | Resource[] | 可见资源点数组（英雄视野范围内的资源） |
| monsters | Monster[] | 可见野兽数组（英雄视野范围内的兽群；仅黑夜有） |

### 1.1.1 Terrain — 地形

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| pos | Pos | 坐标 |
| type | String | 地形类型：`forest` = 密林，`mountain` = 山地矿脉，`water` = 湖泊/河流 |

> **注**：平坦草地（`grassland`）不在此数组中——凡是不在 `terrains` 中的格子即为平坦草地。

### 1.1.2 Resource — 资源点

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| pos | Pos | 坐标 |
| type | String | 资源类型：`wood` / `stone` / `copper_ore` / `iron_ore` / `silicon_ore` / `sulfur` / `berry` / `herb` / `freshwater` |
| remaining | Int | 剩余可采集量（-1 = 无限） |

### 1.1.3 Monster — 野兽

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| id | Int | 野兽唯一编号 |
| pos | Pos | 当前坐标 |
| type | String | 野兽类型：`small` = 小型兽（鼠/狐），`medium` = 中型兽（野猪/狼），`large` = 大型兽（巨蜥），`boss` = BOSS（蚀骨兽王） |
| hp | Int | 当前血量 |
| maxHp | Int | 最大血量 |
| attack | Int | 攻击力 |
| targetType | String | 当前攻击目标类型：`wall` / `tower` / `base` / `hero` / `none`（正在移动中） |
| targetId | Int | 当前攻击目标 ID（无目标时为 -1） |

### 1.1.4 Pos — 坐标

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| x | Int | X 坐标（0-24） |
| y | Int | Y 坐标（0-24） |

## 1.2 Team — 己方队伍信息

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| faction | String | 阵营标识：`A` / `B` |

| gold | Int | 当前剩余金币 |

| buildings | Building[] | 己方全部建筑列表 |
| warehouse | WarehouseSlot[] | 营地仓库物品列表 |
| taskRecords | TaskRecord[] | 任务完成记录 |

### 1.2.1 Hero — 英雄

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| id | Int | 英雄唯一编号 |
| heroType | String | 角色类型：`pioneer` = 开拓者，`gatherer` = 采集英雄，`builder` = 建造英雄 |
| pos | Pos | 当前坐标 |
| hp | Int | 当前血量 |
| maxHp | Int | 最大血量 |
| defense | Int | 防御力 |
| backpack | BackpackSlot[] | 背包物品列表 |
| backpackCapacity | Int | 背包容量上限（开拓者=10，采集/建造=20） |
| alive | Bool | 是否存活（阵亡英雄次日在基地复活） |
| skills | Skill[] | 可用技能列表 |

#### 1.2.1.1 BackpackSlot — 背包物品

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| name | String | 物品名称（如 `wood`, `stone_brick`, `copper_ingot`, `chip` 等，详见 §4 物品名称速查表） |
| quantity | Int | 数量 |

#### 1.2.1.2 Skill — 技能

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| name | String | 技能名称：`move` / `gather` / `build` / `repair` / `synthesize` / `explore` / `use` / `trade` / `operate_tower` |
| available | Bool | 本回合是否可用 |

### 1.2.2 Building — 建筑

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| id | Int | 建筑唯一编号 |
| buildingType | String | 建筑类型：`base` = 基地，`workshop` = 工坊，`warehouse` = 仓库，`research_station` = 研究台，`wall` = 围墙（每段），`defense_tower` = 防御塔 |
| level | Int | 当前等级（1-4） |
| pos | Pos | 建筑坐标（基地为 2×2，该坐标为左下角；其余为 1×1） |
| hp | Int | 当前血量 |
| maxHp | Int | 最大血量 |
| defense | Int | 防御力 |
| attack | Int | 攻击力（仅防御塔有值，其余为 0） |
| attackRange | Int | 射程（仅防御塔有值=2，其余为 0） |
| hasOperator | Bool | 是否有英雄在周围 1 格内操控（仅防御塔有值；true = 本回合可开火） |
| status | String | 状态：`normal` / `damaged`（受损但可用）/ `destroyed`（已摧毁，需重建） |

### 1.2.3 WarehouseSlot — 仓库物品

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| name | String | 物品名称 |
| quantity | Int | 数量 |

### 1.2.4 TaskRecord — 任务记录

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| taskId | String | 任务唯一标识 |
| taskType | String | 任务类型：`causal_reasoning` = 推理类，`long_context` = 长上下文，`web_operation` = Web操作，`file_operation` = 文件操作，`code_bugfix` = 代码bugfix，`db_query` = 数据库检索 |
| status | String | 状态：`in_progress` / `completed` / `failed` / `abandoned` |
| roundAcquired | Int | 接取/触发回合号 |

## 1.3 Enemy — 敌方可见信息

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| faction | String | 阵营标识：`A` / `B` |
| teamId | String | 队伍唯一 ID |
| gold | Int | 已知的对方金币数（仅在中立商人交易中暴露；未暴露时为 -1） |
| totalScore | Int | 对方累计积分 |
| heroes | Hero[] | 视野内可见的敌方英雄（结构与己方 Hero 相同，但不含背包详情——背包不可见） |
| buildings | Building[] | 视野内可见的敌方建筑（结构与己方 Building 相同） |

> **视野规则**：己方英雄周围 **2 格**、己方建筑周围 **2 格** 为视野范围。敌方单位在视野范围内才出现在 `enemy.heroes` 和 `enemy.buildings` 中。

## 1.4 WorldNews — 每日晨间新闻

每天第 1 回合（phase=`day`, roundInPhase=1）推送。其余回合为 null。

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| dayNo | Int | 新闻所属天数 |
| official | String[] | 【官方消息】数组——每条为一个独立情报。包含：突发事件预警（矿井塌方/虫害/火山等）、昨夜兽潮结果统计 |
| folklore | String[] | 【民间传闻】数组——每天 0-2 条。可能包含长上下文任务线索碎片、敌方散布的假消息（无标记，需自行甄别）、纯噪声 |

| 字段 | 数据类型 | 字段说明 |

| smallCount | Int | 今夜小型兽基础数量 |
| mediumCount | Int | 今夜中型兽基础数量 |
| largeCount | Int | 今夜大型兽基础数量 |
| bossCount | Int | 今夜 BOSS 基础数量 |
| extraFromEnemy | ExtraMonster[] | 敌方购买的额外野兽（为空数组表示对方未购买） |

### 1.5.1 ExtraMonster — 敌方购买野兽

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| type | String | 野兽类型：`small` / `medium` / `large` / `boss` |
| count | Int | 数量 |

## 1.7 ActiveTask — 当前激活的自进化任务

当前有未完成的自进化类任务时返回。无激活任务时为 null。

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| taskId | String | 任务唯一标识 |
| taskType | String | 子类型：`web_operation` / `file_operation` / `code_bugfix` / `db_query` |
| description | String | 任务自然语言描述 |
| failCount | Int | 当前已失败次数（达上限后判题器自动放弃本任务） |
| maxFail | Int | 最大允许失败次数（5） |
| taskData | Object | 任务专项数据（结构随 taskType 不同，见下方子表） |

### 1.7.1 taskData 按任务类型

**web_operation：**

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| screenshot | String | 网页截图 Base64 编码（PNG 格式） |
| budget | Int | 可用金币预算 |
| items | WebItem[] | 商品列表（名称+价格±效果，视题目类型可能隐藏部分信息） |
| constraints | String[] | 约束规则文字描述（如"#1与#3互斥""限购2件"等） |

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| id | Int | 商品编号（1-5） |
| name | String | 商品名称 |
| price | Int | 价格（金币）；T5 题目中为 -1 表示"详见规格页" |
| effect | String | 效果描述；T5 题目中为空字符串表示需从子页面获取 |
| tags | String[] | 标签（如 `["攻击"]`, `["防御"]`） |

**file_operation：**

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| sshHost | String | SSH 主机地址（沙盒容器 IP） |
| sshPort | Int | SSH 端口 |
| sshUser | String | SSH 用户名 |
| sshPassword | String | SSH 密码 |
| clue | String | 搜索线索（自然语言描述） |
| totalRounds | Int | 任务可用总回合数 |
| usedRounds | Int | 已消耗回合数 |
| remainingSubmits | Int | 剩余提交次数（初始 5 次） |

**code_bugfix：**

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| sshHost | String | SSH 主机地址 |
| sshPort | Int | SSH 端口 |
| sshUser | String | SSH 用户名 |
| sshPassword | String | SSH 密码 |
| files | String[] | C++ 源文件名列表（2 个） |
| totalBugs | Int | 需要找到的 bug 总数（2 个） |
| remainingSubmits | Int | 剩余提交次数（初始 3 次） |

**db_query：**

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| sshHost | String | SSH 主机地址 |
| sshPort | Int | SSH 端口 |
| sshUser | String | SSH 用户名 |
| sshPassword | String | SSH 密码 |
| question | String | SQL 查询题目（自然语言描述） |
| tableSchema | String | 表结构预览（如 `.schema` 输出） |
| remainingSubmits | Int | 剩余提交次数（初始 3 次） |
| totalRounds | Int | 任务可用总回合数 |
| usedRounds | Int | 已消耗回合数 |

## 1.9 LastRound — 上回合执行结果

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| heroResults | HeroResult[] | 上回合各英雄动作执行结果 |

### 1.9.1 HeroResult

| 字段 | 数据类型 | 字段说明 |
|------|---------|---------|
| heroId | Int | 英雄 ID |
| action | String | 上回合执行的动作 |
| success | Bool | 是否执行成功 |
| code | String | 返回码（见下方各动作返回码表） |
| message | String | 判题器返回的文字描述 |
| returnData | Object | 动作返回的额外数据（结构随 action 不同） |

#### 各动作返回码与 returnData

**move（移动）：**

| code | 含义 | returnData |
|------|------|-----------|
| `success` | 移动到目标格 | `{from: Pos, to: Pos}` |
| `failed:blocked` | 目标格为不可通行地形（水域/山地） | `null` |
| `failed:occupied` | 目标格被其他单位占据 | `null` |
| `failed:out_of_range` | 超出 1 格移动范围 | `null` |
| `failed:dead` | 英雄已阵亡 | `null` |

**gather（采集）：**

| code | 含义 | returnData |
|------|------|-----------|
| `success` | 采集完成，物品已放入背包 | `{item: String, quantity: Int}` |
| `in_progress` | 采集中，还需 N 回合 | `{remainingRounds: Int}` |
| `failed:inventory_full` | 背包已满 | `null` |
| `failed:resource_depleted` | 该格资源已枯竭 | `null` |
| `failed:not_gatherable` | 该格无可采集资源 | `null` |
| `failed:faction_locked` | 该资源为对方阵营独占 | `null` |

**build（建造/升级）：**

| code | 含义 | returnData |
|------|------|-----------|
| `success` | 建造/升级完成 | `{buildingType: String, level: Int, pos: Pos}` |
| `in_progress` | 建造/升级中，还需 N 回合 | `{remainingRounds: Int}` |
| `failed:insufficient_materials` | 背包材料不足 | `{missing: [{item: String, quantity: Int}]}` |
| `failed:tile_occupied` | 目标格已有建筑/围墙 | `null` |
| `failed:prerequisite_not_met` | 前置条件未满足（如研究台未建造、图纸缺失） | `{requirement: String}` |
| `failed:max_count_reached` | 已达该建筑数量上限（防御塔最多 4 座） | `null` |
| `failed:not_at_workshop` | 合成配方需要在工坊周围 1 格内 | `null` |

**repair（维修）：**

| code | 含义 | returnData |
|------|------|-----------|
| `success` | 维修完成，建筑已恢复满血 | `{buildingId: Int, totalHpRestored: Int, totalMaterialsUsed: [{item: String, quantity: Int}]}` |
| `in_progress` | 维修中，本回合已恢复 X HP，还需继续 | `{buildingId: Int, hpRestoredThisTurn: Int, currentHp: Int, maxHp: Int, materialsUsedThisTurn: [{item: String, quantity: Int}]}` |
| `failed:building_undamaged` | 建筑无需维修（已满血） | `null` |
| `failed:insufficient_materials` | 背包材料不足以支付本回合维修 | `{missing: [{item: String, quantity: Int}]}` |

**synthesize（工坊合成）：**

| code | 含义 | returnData |
|------|------|-----------|
| `success` | 合成完成，成品已放入背包 | `{product: String, quantity: Int, materialsConsumed: [{item: String, quantity: Int}]}` |
| `failed:insufficient_materials` | 背包材料不足 | `{missing: [{item: String, quantity: Int}]}` |
| `failed:recipe_locked` | 配方未解锁（工坊等级不够） | `{requiredWorkshopLevel: Int}` |
| `failed:not_at_workshop` | 建造英雄不在工坊周围 1 格内 | `null` |

**explore（探索）：**

| code | 含义 | returnData |
|------|------|-----------|
| `success` | 探索完成 | `{description: String}` — 判题器以自然语言描述当前坐标附近发现的内容。若在任务官(13,10)处 → 返回可接任务信息；若在(12,12)处 → 返回石坛相关信息；若在中立商人处 → 返回当前各资源买卖价格+剩余配额 |
| `failed:dead` | 英雄已阵亡 | `null` |

**use（使用物品/操作终端）：**

| code | 含义 | returnData |
|------|------|-----------|
| `success` | 使用/操作完成 | 视使用场景不同（见下方子表） |
| `failed:no_item` | 背包中无此物品 | `null` |
| `failed:wrong_location` | 当前位置无法执行此操作 | `null` |

use 动作的 returnData 按场景：

| 场景 | returnData |
|------|-----------|
| 使用消耗品（药膏等） | `{itemUsed: String, effect: String, hpRestored: Int}` |
| 网页终端操作（click/scroll） | `{screenshot: String（Base64 PNG）, action: String, result: String}` |
| SSH 终端 shell 命令 | `{command: String, stdout: String, stderr: String}` |
| 提交任务答案 | `{result: String（"correct"/"incorrect"）, detail: String}` |
| 石坛放置祭品 | `{result: String, message: String}` |

**trade（贸易）：**

| code | 含义 | returnData |
|------|------|-----------|
| `success` | 交易完成 | `{tradeType: String, given: TradeItem, received: TradeItem}` |
| `failed:no_trade_request` | 无待响应的交易请求 | `null` |
| `failed:insufficient_items` | 己方物品不足 | `{missing: [{item: String, quantity: Int}]}` |
| `failed:insufficient_gold` | 己方金币不足 | `{shortfall: Int}` |
| `failed:merchant_daily_limit` | 中立商人每日买卖限额已满（各 10 单位） | `null` |

| code | 含义 | returnData |
|------|------|-----------|
| `success` | 正在操控防御塔 | `{towerId: Int, towerPos: Pos}` |

# 2 响应报文（Response）顶层结构

选手 → 判题器。每回合返回一次。

| 字段 | 数据类型 | 是否必选 | 说明 |
|------|---------|---------|------|
| heroCommands | HeroCommand[] | M（必填） | 3 名英雄的本回合动作指令数组（阵亡英雄也需返回，action 填 `nothing`） |

## 2.1 HeroCommand — 单英雄指令

| 字段 | 数据类型 | 是否必选 | 说明 |
|------|---------|---------|------|
| heroId | Int | M（必填） | 英雄 ID |
| action | String | M（必填） | 执行动作标识（见下方取值表） |
| targetPos | Pos | 条件必填 | 目标坐标（move / gather / build / repair / explore / use / operate_tower 必填） |
| buildType | String | 条件必填 | 建造/升级目标建筑类型（build 动作必填）：`wall` / `defense_tower` / `workshop` / `warehouse` / `research_station` |
| buildLevel | Int | 条件必填 | 建造/升级目标等级（build 动作必填；新建=1，升级=2/3/4） |
| recipe | String | 条件必填 | 合成配方名称（synthesize 动作必填），如 `wooden_plank`, `stone_brick`, `copper_ingot`, `iron_ingot`, `pure_silicon`, `gunpowder`, `ointment`, `pure_iron`, `transistor`, `circuit_board`, `chip` |
| taskAnswer | String | 条件必填 | 任务作答内容（提交任务答案时必填） |
| queryInfo | String | 条件必填 | 入参数据。SSH 终端命令（文件操作/代码bugfix/数据库检索）/ SQL 语句（数据库检索） |
| tradeResponse | TradeResponse | 条件必填 | 交易响应（仅当存在 tradeRequest 且需响应时填写） |
| useAction | String | 条件必填 | use 动作的子类型：`click` / `scroll` / `command` / `submit_answer` / `use_item` |
| useData | UseData | 条件必填 | use 动作的附加数据（结构随 useAction 不同） |
| warehouseAction | String | 非必填 | 仓库操作（仅当英雄位于仓库周围 1 格内有效）：`deposit` = 存入，`withdraw` = 取出 |
| warehouseItem | String | 条件必填 | 仓库操作目标物品名称 |
| warehouseQuantity | Int | 条件必填 | 仓库操作目标数量 |

### action 取值表

| action | 说明 | 可用英雄 | 可用阶段 |
|--------|------|---------|---------|
| `nothing` | 无操作（驻留原地） | 全部 | 全天 |
| `move` | 向 targetPos 移动 1 格 | 全部 | 全天 |
| `gather` | 在当前位置采集资源 | 采集英雄 | 仅白天 |
| `build` | 在 targetPos 建造/升级建筑 | 建造英雄 | 仅白天 |
| `repair` | 维修 targetPos 处的建筑（每回合恢复 10% 最大HP，材料按同比例累积扣除。维修期间不能进行任何其他操作） | 建造英雄 | 全天 |
| `synthesize` | 在工坊合成物品 | 建造英雄 | 仅白天 |
| `explore` | 探索当前位置周围 | 开拓者 | 仅白天 |
| `use` | 使用物品/操作终端/提交答案 | 开拓者 | 仅白天 |
| `trade` | 在中立商人处发起/响应交易 | 开拓者 | 仅白天 |
| `operate_tower` | 操控 targetPos 处的防御塔 | 全部 | 仅黑夜 |

### 任务提交格式（taskAnswer）

| 任务类型 | taskAnswer 格式 | 示例 |
|---------|---------------|------|
| Web操作 | 所选商品编号，逗号分隔 | `"1,4"` |
| 文件操作 | 密钥字符串 | `"SxK7mP...Pf"` |
| 代码bugfix | `文件名 行号 bug类型` | `"transmitter.cpp 18 null_pointer"` |
| 数据库检索 | 查询结果（按题目要求的格式） | `"3|1250.5"` |
| 长上下文（石坛宝藏） | 三样物品依次放置 | `"stone_brick,iron_ingot,gunpowder"` |

### 2.1.1 TradeResponse — 交易响应

| 字段 | 数据类型 | 是否必选 | 说明 |
|------|---------|---------|------|
| accept | Bool | M（必填） | 是否接受对方交易请求 |
| counterOffer | TradeItem | 非必填 | 还价（仅当 accept=false 且想提出替代方案时填写） |

### 2.1.2 UseData — use 动作附加数据

按 useAction 取值：

**click（网页点击）：**

| 字段 | 数据类型 | 说明 |
|------|---------|------|
| coordinate | Pos | 点击坐标（相对于截图左上角的像素坐标） |

**scroll（网页滚动）：**

| 字段 | 数据类型 | 说明 |
|------|---------|------|
| direction | String | 滚动方向：`up` / `down` |
| length | Int | 滚动像素距离 |

**command（SSH 终端命令）：**

> 使用 HeroCommand.queryInfo 字段传入命令字符串

**submit_answer（提交任务答案）：**

> 使用 HeroCommand.taskAnswer 字段传入答案

**use_item（使用消耗品）：**

| 字段 | 数据类型 | 说明 |
|------|---------|------|
| itemName | String | 物品名称（如 `ointment`） |
