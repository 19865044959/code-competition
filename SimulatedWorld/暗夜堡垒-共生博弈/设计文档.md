## 一、游戏总览

### 1.1 核心概念

两名玩家在 25×25 方格地图上各控制 **3 名英雄**（开拓者×1 + 采集英雄×1 + 建造英雄×1），白天采集建造、执行 Agent 任务、外交贸易，夜晚必须回防操控防御塔抵御递增强度的兽潮。10 天（2000 回合）后按累计积分定胜负。

地图资源分布不均——**铜矿只有A能采、硅矿只有B能采**（阵营资源绑定），中立商人是获取对方独占资源的唯一途径。芯片=铜+硅，是满级建筑的必须材料。不合作=造不出芯片=永远卡在Lv3=后期兽潮无法抗衡。

**金币经济体系**让 Agent 通过情报推演预判价格波动进行套利，通过抢占配额逼迫对手高价交易，通过金币购买野兽增加对方防守压力。

### 1.2 对局基础配置

| 参数 | 数值 |
|------|------|
| 对局模式 | 1V1，双方出生点地图对角对称分布 |
| 对局周期 | 10 游戏天 |
| 回合概念 | **1 天 = 200 回合**（白天 120 回合，黑夜 80 回合），10 天 = 2000 回合 |
| 回合定义 | 每回合 = 判题器发送当前地图信息 → 玩家收到后返回本回合动作 → 判题器执行并反馈结果 |
| 每队英雄数 | 3 名（**开拓者×1、采集英雄×1、建造英雄×1**） |
| 英雄阵亡 | 次日在基地复活 |
| 淘汰条件 | 基地被摧毁 → 直接淘汰，后续无法获取积分 |
| 地图规格 | 25×25 正方形网格 |
| 双方关系 | **竞争&合作**——不可直接攻击对方，通过贸易互换材料+金币博弈+积分排名竞争 |

### 1.3 三种英雄角色

| 角色 | 数量 | 核心职责 |
|------|:---:|------|
| **开拓者** | 1 | 贸易谈判、执行 Agent 任务（探索/使用）、夜间辅助操控防御塔 |
| **采集英雄** | 1 | 采集所有资源、夜间操控防御塔 |
| **建造英雄** | 1 | 建造/升级建筑、维修建筑、工坊合成、夜间操控防御塔 |

### 1.4 核心循环

```
【晨间】
  → 系统推送当日新闻（含【官方消息】、【民间传闻】）
  → 玩家查看新闻、盘点库存、检查建筑状态
  → 昨夜阵亡英雄在基地复活

【白天阶段】（120 回合）
  ├── 采集英雄：采集资源
  ├── 建造英雄：建造升级、维修建筑、工坊合成
  ├── 开拓者：外出执行 Agent 任务、中立商人交易
  ├── 在中立商人处进行资源⇄金币交易（配额共享，先到先得）
  └── 黄昏：黑夜临近预警通知

【黑夜阶段】（80 回合）
  ├── 兽潮从四面八方来袭，每夜递增
  ├── 防御塔需要英雄站在旁边（周围1格）操控才能开火（射程2格）
  ├── 留在野外未回防的英雄 → 被野兽攻击，极大概率阵亡
  ├── 英雄阵亡 → 次日在基地复活
  ├── 围墙阻挡兽群前进
  ├── 天亮 → 阳光彻底杀死地图上所有残余兽群
  ├── 建筑受损 → 建造英雄可紧急维修，但维修期间不能做任何其他操作，优先在黄昏前修好
```

### 1.5 10 天节奏

| 阶段 | 天数 | 每夜兽潮 | 核心主题 |
|------|------|---------|---------|
| 新手期 | 第 1 天 | 2-3 只小型兽 | 搭建基础防御、熟悉操作 |
| 发育期 | 第 2-5 天 | 递增至 8-10 只混合兽 | 升级建筑、执行 Agent 任务、情报套利、初步外交 |
| 中期 | 第 6-8 天 | 10-15 只混合+大型兽 | 获取图纸升级满级建筑、竞争加剧、配额抢占博弈 |
| 末期 | 第 9-10 天 | 15-25 只+BOSS | 极限防守、最终博弈、冲刺积分、宝藏开启 |

## 二、地图与地形系统

### 2.1 地图规格

- 全图 **25×25** 标准正方形网格
- 单个地形格子为 1×1 通行单元
- **基地** 占用 **2×2** 网格，其余建筑（研究台、工坊、仓库、围墙、防御塔）占用 **1×1** 网格
- 围墙为 **1×1** 分段
- 两名玩家出生点位于地图对角对称位置

### 2.2 地形分类

| 地形类型 | 网格占比 | 核心资源产出 |
|---------|---------|------------|
| 平坦草地 | 30% | 浆果 |
| 密林区域 | 20% | 木材、草药 |
| 山地矿脉 | 40% | 石料、铜矿、铁矿、硅矿、硫磺 |
| 湖泊/河流 | 10% | 淡水 |
| 任务官 | 固定坐标(13,10)（1 处） | 自进化类任务接取（Web操作/文件操作/代码bugfix/数据库检索） |
| 中立商人 | 地图正中心（1 处） | 玩家间贸易 + 资源买卖 + 野兽召唤/假消息散布——所有经济活动的统一入口 |

### 2.3 阵营资源绑定

- **无领地概念**。地图资源对双方英雄可见、可到达
- **阵营资源绑定**：
  - **铜矿**：只有 **A 阵营**英雄可以采集
  - **硅矿**：只有 **B 阵营**英雄可以采集
  - 其余资源（木材、石料、铁矿、硫磺、食物、草药、淡水）：双方均可采集
- 对方独占资源只能通过**中立商人**交换获得
- 所有己方建筑和英雄共享视野
- 每个建筑自带基础视野（周围 2 格）

### 2.4 资源分布不对称

| 资源类型 | 玩家 A | 玩家 B | 说明 |
|---------|--------|--------|------|
| 木材 | ★★★★★ | ★★★★★ | **双方均丰富** |
| 石料 | ★★★★☆ | ★★★★☆ | 双方均有，建筑基础材料 |
| 铁矿石 | ★★★★☆ | ★★★★☆ | **双方均可** → 铁锭 → Lv3 围墙/防御塔 |
| 硫磺 | ★★★☆☆ | ★★★☆☆ | **双方均有** → 火药（防御塔 Lv3 需要） |
| 铜矿石 | ✅ **只有A能采** | ❌ | A 独占 → 铜锭 → 晶体管/电路板 → 芯片 |
| 硅矿石 | ❌ | ✅ **只有B能采** | B 独占 → 纯硅 → 电路板 → 芯片 + 激光塔 |
| 食物（浆果） | ★★★★★ | ★★☆☆☆ | A 供食物 |
| 草药 | ★★☆☆☆ | ★★★★★ | B 供草药 → 药膏（高级回血品） |

**强制合作锁——芯片链**：
```
芯片 = 晶体管 + 电路板
     = (铜锭 + 铁锭) + (铜锭 + 纯硅)
          ↑A独占  ↑共有     ↑A独占  ↑B独占

→ 铜矿只有A能采、硅矿只有B能采（阵营资源绑定，走到面前也采不了）
→ 铁锭/硫磺双方自给、铜锭只有A能产、纯硅只有B能产
→ 芯片 = 必须通过中立商人交换才能获得的复合产物
→ 没有芯片 = 永远卡在Lv3 = 后期兽潮无法抗衡（第8晚起激光塔/合金城墙刚需）
```

**渐进式合作节奏**：
```
Lv1→Lv2：纯石料/木材（完全自给，不需要合作）
Lv2→Lv3：石料 + 铁锭 + 火药（铁中立、硫磺双方均有→纯自给）
Lv3→Lv4：芯片 + 纯铁 + 纯硅（芯片横跨双方，强制深度合作）
```

## 三、英雄系统

### 3.1 基础规则

- 每名玩家控制 **3 名英雄**，各司其职：
  - **开拓者 ×1** — 贸易谈判、执行 Agent 任务（探索/使用）、夜间辅助操控防御塔
  - **采集英雄 ×1** — 资源采集、夜间操控防御塔
  - **建造英雄 ×1** — 建筑建造/升级、建筑维修、工坊合成、夜间操控防御塔
- 英雄阵亡后在己方**基地**次日起复活（夜间阵亡 → 次日清晨复活）
- 每个英雄有独立背包
- 双方英雄**不可互相攻击**（竞争性合作）
- **英雄无攻击能力**——所有英雄不能攻击野兽或其他单位，只能依靠防御塔和城墙防守。英雄被野兽攻击时只能承受伤害或逃离
- **每回合所有英雄移动 1 格**
- 英雄**无等级系统**，属性固定不变

### 3.2 开拓者

**核心定位**：贸易谈判、执行 Agent 任务

| 属性 | 数值 |
|------|------|
| HP | 130 |
| 防御力 | 6 |
| 背包容量 | 10 格 |

**核心技能**：

| 技能 | 消耗 | 效果 |
|------|------|------|
| **探索** | 1 回合 | 深度探明当前坐标周围区域隐藏内容：任务线索、古代终端等。判题器以自然语言或系统接口返回探索结果 |
| **使用** | 1 回合 | 在指定坐标使用背包中的物品，或操作终端（点击/滚动/输入） |
| **贸易** | 1 回合 | 在中立商人处发起/响应交易请求。支持物⇄物和金币⇄物两种模式 |

**核心职责**：

- 唯一可执行 Agent 任务中"探索"和"使用"动作的英雄
- 唯一可发起玩家间贸易的英雄
- 唯一可访问古代终端（文件系统挖掘任务）和网页终端（网页任务）的英雄
- 不参与采集、建造
- 夜间也可操控防御塔

### 3.3 采集英雄

**核心定位**：资源采集、夜间操控防御塔

| 属性 | 数值 |
|------|------|
| HP | 150 |
| 防御力 | 8 |
| 背包容量 | 20 格 |

**核心技能**：

| 技能 | 消耗 | 效果 |
|------|------|------|
| **采集** | 1 回合 | 在资源地形格上执行采集，持续至产出完成。完成后物品自动放入背包 |

**核心职责**：

- 采集所有类型资源（包括阵营独占资源：A采铜、B采硅）
- **夜晚站在防御塔旁（1 格内）操控防御塔攻击兽群**

### 3.4 建造英雄

**核心定位**：建筑建造/升级、建筑维修、工坊合成、夜间操控防御塔

| 属性 | 数值 |
|------|------|
| HP | 150 |
| 防御力 | 8 |
| 背包容量 | 20 格 |

**核心技能**：

| 技能 | 消耗 | 效果 |
|------|------|------|
| **建造** | 持续 N 回合 | 在空地格上建造/升级建筑，中途不可离开。材料在完成时从背包扣除 |
| **维修** | 持续 N 回合 | 维修受损建筑。每回合恢复最大 HP 的 10%，材料按同比例消耗（每回合扣建造材料总量的 10%）。可随时中断，已恢复 HP 保留。**维修期间不能进行任何其他操作** |
| **合成** | 1 回合 | 在工坊将材料合成为成品。需建造英雄站在工坊周围 1 格内，材料从背包扣除，成品放入背包 |

**核心职责**：

- 建造所有建筑、升级建筑
- 维修所有受损建筑（按血量比例消耗材料）
- 在工坊合成所有物品（**唯一可执行合成的英雄**）
- **夜晚站在防御塔旁（1 格内）操控防御塔攻击兽群**

### 3.5 贸易规则

- 开拓者走到地图中心**中立商人** → 发起交易请求
- 对方开拓者**不需要在场**（远程响应）
- 交易界面显示对方当前各项资源库存量
- **两种交易模式**：
  - **物⇄物**：双方确认交换清单 → 各自仓库物品隔空瞬间交换
  - **金币⇄物**：双方协商金币定价 → 金币和物品隔空交换
- 中立商人不可被摧毁、不可被占据

### 3.6 背包与仓库容量

| 容器 | 容量 |
|------|------|
| 开拓者背包 | 10 格 |
| 采集英雄背包 | 20 格 |
| 建造英雄背包 | 20 格 |
| 营地仓库 | 80 格（Lv2 → 120 格，Lv3 → 200 格） |

> 仓库存取规则：英雄站在仓库周围 1 格内，可向判题器请求存取物品。

### 3.7 建筑维修规则

建造英雄可维修受损的建筑（围墙、防御塔、工坊、仓库、研究台）。**白天与夜间统一规则**：

**维修速率**：每回合恢复最大 HP 的 **10%**（向上取整到整数 HP）。

**材料消耗**：每回合消耗建造材料总量的 **10%**（与 HP 恢复比例一致）。由于 10% 可能产生小数，判题器内部以浮点数累积，当某材料累积值 ≥ 1 时从背包扣除 1 个并减 1。维修结束时有未满 1 的累积值向上取整扣除。

```
每回合恢复 HP = ceil(maxHP × 10%)
每回合材料累积 = 建造材料总量 × 10%
背包扣除时机：累积值 ≥ 1 时扣 1 个；结束时剩余累积向上取整
```

> **小数示例**：围墙 Lv1 建造材料为木材×2。每回合累积 2×0.1=0.2 木材。第 5 回合累积值首次达到 1.0 → 背包扣 1 木材，累积值归 0。第 10 回合又累积到 1.0 → 再扣 1。总消耗 2 木材，与 100% 修复量匹配。

维修为**持续动作**——建造英雄驻留维修，每回合自动恢复对应 HP 并扣除对应材料，直至建筑满血或主动中断。**维修期间不能进行任何其他操作**（移动/建造/合成/操控防御塔等均不可）。

**所有建筑统一 10 回合从 0 到满血**：

| 当前 HP 比例 | 至满血需 | 说明 |
|:---:|:---:|------|
| 0%（濒毁） | 10 回合 | 最坏情况 |
| 50% | 5 回合 | 黄昏时半血需抓紧修 |
| 80% | 2 回合 | 轻微受损 |
| 90% | 1 回合 | 几乎完好 |

**按建筑等级的每回合恢复量**：

| 建筑等级 | 围墙 HP/回合 | 防御塔 HP/回合 | 工坊 HP/回合 | 仓库 HP/回合 |
|:---:|:---:|:---:|:---:|:---:|
| Lv1 | +10 | +20 | +35 | +35 |
| Lv2 | +30 | +35 | +40 | +40 |
| Lv3 | +60 | +50 | +50 | +50 |
| Lv4 | +100 | +80 | — | — |

> **设计意图**：统一百分比规则带来了几个好处——① 白天不再是"瞬间修好"，建筑受损后无论昼夜都需要投入回合，强化了防守规划的重要性；② 高级建筑每回合恢复绝对值更大（Lv4 围墙每回合回 100 HP），自然奖励升级投入；③ 维修期间建造英雄完全被占用——无论昼夜，选择维修就意味着放弃该时间段内的建造/合成/控塔等其他一切操作，这是一个需要权衡的机会成本决策。

### 3.8 采集与建造动作

#### 3.8.1 采集动作

采集英雄可在资源地形格上执行"采集"动作。

| 返回码 | 含义 | Agent 后续动作 |
|--------|------|---------------|
| `success` | 采集完成，物品已放入背包 | 继续采集或移动 |
| `in_progress` | 采集中，还需 N 回合 | 驻留继续 |
| `failed:inventory_full` | 背包满，物品无法放入 | 回库存放或丢弃物品 |
| `failed:resource_depleted` | 该格资源已枯竭 | 换位置采集 |
| `failed:not_gatherable` | 该格无可采集资源 | 换地形 |
| `failed:faction_locked` | 该资源对方独占（铜/硅） | 放弃，只能贸易获取 |

#### 3.8.2 建造动作

建造英雄可在空地格上执行"建造"动作，持续 N 回合完成。

建造限制：
- **不可在已有建筑/围墙的格子上建造**
- 除此之外**无任何位置限制**——全图可建，不限制离基地距离，不限制中线

| 返回码 | 含义 | Agent 后续动作 |
|--------|------|---------------|
| `success` | 建造完成，建筑已落成 | 下一个动作 |
| `in_progress` | 建造中，还需 N 回合 | 驻留继续 |
| `failed:insufficient_materials` | 材料不足 | 先采集材料 |
| `failed:tile_occupied` | 目标格已有建筑/围墙 | 重新选位置 |

## 四、夜间防守系统

### 4.1 防御塔操控规则

```
白天：
  防御塔不工作（野兽不出没）

黑夜（80 回合）：
  兽潮从四面八方涌来
  最多 4 座防御塔，3 名英雄 → 英雄需不断换位操控不同方向的塔
  有英雄站在周围 1 格内 → 塔以面板伤害攻击射程内最近目标
  无英雄操控 → 塔不攻击

黎明：
  → 阳光彻底杀死地图上所有残余兽群
  → 阵亡英雄在基地复活
```

## 五、建筑系统

> 总共 **6 种建筑**：基地、研究台、工坊、仓库、围墙、防御塔

### 5.1 建筑分类

| 分类 | 建筑 | 可否被摧毁 |

### 7.4 兽潮行为逻辑

**核心原则：野兽的目标是基地，不是沿途单位。**

#### 规则一：路径优先（Path Priority）

野兽沿最短路径向基地移动。只要能前进就绝不攻击——哪怕路过防御塔旁边被射击。只有被**物理阻挡**（前方格子有建筑/围墙）时才停下来攻击阻挡物。

#### 规则二：阻挡触发仇恨（Block → Aggro）

当野兽无法前进时，攻击阻挡物以清出道路。仇恨**绑定在该阻挡物上**，直到阻挡物被摧毁，然后重新寻路攻击下一个阻挡物。

#### 规则三：仇恨转移条件

| 触发条件 | 行为 |
|---------|------|
| 当前阻挡物被摧毁 | 重新寻路，攻击路径上下一个阻挡物 |
| 野兽路径被挡，邻格有英雄 | 野兽攻击英雄（英雄无攻击能力，只能承受伤害或逃离） |
| 野兽1格内**同时有基地和英雄** | 野兽**直接攻击基地** |

#### 规则四：远程攻击不拉仇恨

防御塔（射程2格）远程攻击野兽时，野兽**不转移仇恨**。

#### 补充规则

- 防御塔需要英雄站在旁边（1 格内）操控才攻击
- **所有建筑均可被兽群破坏**
- 黎明阳光杀死所有残余兽群
- **一方被淘汰后，兽潮数量减半**

## 八、Agent 任务系统

本游戏是**以游戏为壳的 Agent 能力评测平台**。共设计 **3 类 Agent 任务**（自进化类含多种子类型），分为两条通道。详细设计见 `主要参考/agent_task_system_design.md`。

### 8.1 双通道架构

| 通道 | 任务类型 | 交互方式 | 特点 |
|------|---------|---------|------|
| **🏛️ 任务官** | 自进化类（Web操作 / 文件操作 / 代码bugfix 等） | 前往任务官主动接取 | 需与外部系统交互（网页/终端） |
| **📰 世界新闻** | 推理类、长上下文 | 每天晨间新闻自动推送，被动接收 | 纯信息输入，可并行处理 |

### 8.2 任务总览

| 任务类型 | 考察维度 | 触发方式 | 说明 |
|---------|---------|---------|------|
| **推理类（官方消息）** | 因果链推理、价格预判、配额抢占、双边谈判 | 📰 晨间新闻【官方消息】自动推送；中立商人交互 | 情报推演→套利+策略调整；双边贸易谈判 |
| **长上下文（民间传闻）** | 跨天记忆、碎片拼凑、物品映射 | 📰 晨间新闻【民间传闻】每天推送碎片 | 4条碎片分散在7天中，集齐后拼凑完整线索→到指定地点按顺序使用物品 |
| **自进化类 · Web操作** | OCR视觉理解、约束优化决策 | 🏛️ 任务官分配 → 返回网页截图（PNG） | 浏览网页、截图理解、做出购买决策 |
| **自进化类 · 文件操作** | Shell命令使用、信息搜索、线索解读、自进化 | 🏛️ 任务官分配 → 返回 SSH 连接+搜索线索 | 在沙盒Linux中搜索密钥→开启宝箱→获得奖励 |
| **自进化类 · 代码bugfix** | C++ 代码阅读、逻辑推理、bug 精准定位 | 🏛️ 任务官分配 → 返回 SSH 连接（2个C++文件） | 纯阅读定位bug，提交文件名+行号+类型→获得奖励 |
| **自进化类 · 数据库检索** | SQL语法、条件理解、聚合函数、查询正确性 | 🏛️ 任务官分配 → 返回 SSH 连接（SQLite沙盒） | 判题器出题，Agent 写 SQL 查询返回结果 |
| **自进化类 · （后续子类型）** | 待定 | 🏛️ 任务官分配 | 子类型持续增加，从池中随机抽取 |

### 8.3 任务官系统

#### 8.3.1 定位

"任务官"统一管理所有**自进化类**任务。开拓者前往**任务官**（固定独立坐标，坐标(13,10)）与任务官对话，接取本局唯一的自进化任务。

#### 8.3.2 核心机制

```
任务官坐标：固定独立坐标（坐标(13,10)）
     ↓
开拓者到达 → [探索] → 判题器告知本局自进化任务类型（从子类型池中随机抽取 1 种，整局不变）
     ↓
开拓者确认接取 → 判题器分配该类型的第 1 道题目
     ↓
完成 → 刷新同类型的下一道题目（难度递进或场景更换）
     ↓
该类型全部题目完成 → 任务官提示"当前无可接任务"
     ↓
单道题目累计失败 5 次 → 自动放弃该题，刷新下一道（如有剩余）
```

#### 8.3.3 规则

| 规则 | 说明 |
|------|------|
| 可用时间 | 无限制 |
| 任务类型 | 每局从子类型池（**Web操作 / 文件操作 / 代码bugfix / 数据库检索**，后续持续增加）中**随机抽取 1 种**，整局不变 |
| 题目数量 | 该类型下含 **多道题目**（如 Web操作有 T1/T2/T5 三级难度，代码bugfix有 3 组不同文件/bug，文件操作有 3 种线索类型，数据库检索有 3 道不同 SQL 题） |
| 刷出方式 | 逐题刷出，完成/放弃一道后刷下一道，难度递进或场景更换 |
| 失败上限 | 每道题目**最多累计失败 5 次** → 自动放弃该题，刷下一道 |
| 设计意图 | 同一类型跨对局反复出现 → Agent 总结该类型的 SOP → 后续对局中同类型任务表现越来越好。同一类型内的多道题目保证 Agent 不是靠背答案，而是真正掌握该类型的解题范式 |

### 9.1 双轨经济体系

│                    经济双轨制                         │
├──────────────────┬──────────────────────────────────┤
│  玩家 ↔ 玩家       │       玩家 ↔ 中立商人（资源买卖）   │

│                    │  → 情报推演 → 预判价格 → 套利     │
└──────────────────┴──────────────────────────────────┘

### 9.2 金币体系

**金币唯一来源**：在中立商人处进行资源买卖。
- 在中立商人处卖资源 → 获得金币
- 在中立商人处用金币买资源 → 消耗金币
- 金币不随时间增长、不打怪掉落、不完成任务直接获得

**金币用途**：
1. **Web 商店买装备**（Web 操作任务）
2. **购买野兽召唤令**攻击对方（见第十一章）
3. **向对手高价买资源**（中立商人配额被抢占后的替代渠道）
4. **散布假消息**干扰对方（见第十一章 §11.4）

### 9.3 中立商人：定价与配额

每种物品独立定价，每天买卖配额各 10 单位，双方共享，先到先得：

| 资源 | 基准买卖价 | 每日配额 | 价格波动触发 |
|------|:---:|:---:|------|
| 木材 | 1 金币/单位 | 各 10 | 虫害→涨价至 1.5 |
| 石料 | 1 金币/单位 | 各 10 | — |
| 铁矿石 | 1.5 金币/单位 | 各 10 | 矿井塌方→涨价至 3.0 |
| 硫磺 | 1.5 金币/单位 | 各 10 | 火山预警→涨价至 3.0 |
| 铜矿石 | 2 金币/单位 | 各 10 | 仅 A 可采→A 有定价权 |
| 硅矿石 | 2 金币/单位 | 各 10 | 仅 B 可采→B 有定价权 |
| 淡水 | 0.5 金币/单位 | 各 10 | 干旱→涨价 |
| 草药 | 1 金币/单位 | 各 10 | — |
| 食物（浆果） | 0.5 金币/单位 | 各 10 | — |

> 价格波动：中立商人价格随突发事件实时调整，不提前公告。Agent 需从【官方消息】推理判断价格走向。

## 十、对抗系统

### 10.1 对抗手段总览

```
┌─────────────────────────────────────────────┐
│                 玩家间对抗                     │
├──────────────┬──────────────────────────────┤
│  间接对抗     │         直接干扰              │
├──────────────┼──────────────────────────────┤
│ ① 商人谈判    │ ③ 金币购买野兽攻击对方         │
│   议价博弈    │   下一晚对方多一只对应野兽       │
│              │                              │
│ ② 中立商人   │ ④ 散布假消息干扰对方            │
│   配额抢占    │   混入对方民间传闻，扰乱长上下文   │
│   → 逼对手    │   推理                        │
│     高价交易  │                              │
└──────────────┴──────────────────────────────┘
```

### 10.2 金币购买野兽

开拓者可在中立商人处使用金币购买野兽召唤令，**下一晚对方兽潮额外增加对应野兽**。具体物品及价格见 §9.4 中立商人额外服务。

**设计约束**：
- 每天最多购买 **2 次**（防止极端堆叠）
- 购买的野兽叠加在对方当夜正常的兽潮之上
- 仅影响下一晚，不跨夜累积
- 对方在晨间新闻【官方消息】中会看到额外数量，并标注"（来自敌方购买）"

### 10.3 博弈场景

```
第 9 晚对方已有 1 BOSS：
  → 你再买 1 BOSS 召唤令 → 对方当夜面对 2 BOSS
  → 防线可能崩溃 → 基地被毁 → 直接淘汰
  → 你付出的金币 vs 对方可能的淘汰 = 高风险高回报

防守方对策：
  → 发现对方买了野兽 → 紧急加固防御
  → 或者：你也买野兽反击 → 形成军备竞赛
```

### 10.4 散布假消息

开拓者可在中立商人处花费 **50 金币** 散布一条假消息，输入任意文本（不限内容、不审查），判题器将其原样混入对方**下一天**的【民间传闻】中，**不做任何标记**。

| 规则 | 说明 |
|------|------|
| **触发方式** | 开拓者访问中立商人 → 花费 50 金币 → 输入任意文本 |
| **频率限制** | 每名玩家**每局最多 1 次** |
| **投递时机** | 当天购买，对方下一天晨间新闻的【民间传闻】中收到 |
| **对方感知** | 假消息与真实【民间传闻】格式完全一致，无标记、无来源标注 |
| **内容限制** | 无。选手自行决定输入什么，判题器不做审查 |

**设计意图**：这是针对长上下文任务的定向干扰——对方需要在混杂了真实线索、自然噪声、以及可能存在的敌方假消息的传闻流中，甄别信号、过滤干扰、拼凑真相。假消息不影响游戏数据（价格、资源等），仅增加对方推理成本。

## 十一、新闻与预警系统

### 11.1 每日晨间新闻（2 频道结构）

每天清晨判题器推送晨间新闻，按频道分类，Agent 根据频道标签决定处理策略：

```
═══════════════════════════════════════════════════════════
  📰 第 N 天晨间新闻
═══════════════════════════════════════════════════════════

【官方消息】           ← 每天必读
【民间传闻】           ← 每天 0-1 条，可能混入对方散布的假消息（无标记）
═══════════════════════════════════════════════════════════
```

| 频道 | 频率 | 包含内容 | 对应任务 | 内容示例 |
|------|:---:|------|:---:|------|
| 【官方消息】 | 每天 | ① 兽潮预警（今晚兽潮种类+数量，含敌方购买标注）② 时间通报（黑夜/黄昏/黎明/白天切换）③ 突发事件预警（如矿井塌方事故导致铁矿产区受损） | 推理类任务（情报推演+双边博弈） | "北部铁矿区发生矿井塌方，预计约1天后影响开采，持续约2天" |
| 【民间传闻】 | 每天 0-1 条，可额外叠加敌方假消息 | NPC口述 + 可能混入对方散布的假消息（无标记，需自行甄别） | 长上下文任务（碎片拼凑 + 噪声过滤） | 真实线索与假消息混杂，Agent 自行辨别 |

- **自进化类任务（Web操作/文件操作/代码bugfix）**：通过任务官统一分配，不直接出现在晨间新闻中

### 11.2 价格波动通知

中立商人价格随【官方消息】中的突发事件实时调整，**不单独公告**。Agent 需从情报中自主推理价格走向。当前价格可通过开拓者访问中立商人查询。

## 十二、积分与排名系统

### 12.1 积分获取

| 行为 | 积分 | 说明 |
|------|------|------|
| 熬过 1 晚 | +10×天数 | 第 N 天天亮：+10N 分 |
| 击杀小型兽 | +1 | 每只 |
| 击杀中型兽 | +2 | 每只 |
| 击杀大型/精英兽 | +4 | 兽潮 |
| 击杀 BOSS | +10 | 后期夜晚 |
| **完成推理任务（推理类）** | — | 奖励为情报套利获得的金币 |
| **完成长上下文任务** | **+60** | 成功开启宝藏 |
| **完成自进化类任务** | **+30** | Web操作/文件操作/代码bugfix/数据库检索统一；首次提交正确额外获得合金城墙图纸×1 |

### 12.2 淘汰机制

- 如果**基地**被兽潮摧毁 → 玩家**直接淘汰**
- 淘汰后果：
  - 无法继续获取任何积分（已获得的积分保留）
  - 剩余天数不再参与，直接观战
  - 对方后续夜晚只承受自己一侧的兽潮

### 12.3 全局排名机制

- 每局 10 天结束后，双方各自累积的积分为**本局最终得分**
- 积分计入**全服跨对局大排行榜**（赛季制，如 1 个月/赛季）
- 积极合作贸易 → 双方都高分 → 大榜排名双高
- 积极完成 Agent 任务 → 获取图纸+积分，拉开排名差距
- 排名奖励在赛季末结算

### 13.2 每天流程

```
【晨间】
  → 系统推送当日新闻（【官方消息】/【民间传闻】）
  → 玩家查看新闻、盘点库存、检查建筑状态
  → 昨夜阵亡英雄在基地复活

【白天 — 120 回合】
  → 采集英雄：采集资源
  → 建造英雄：建造建筑 / 维修建筑 / 工坊合成
  → 开拓者：执行 Agent 任务 / 访问中立商人 / 访问任务官

【黄昏】
  → 系统推送今夜兽潮预警（含敌方购买野兽标注）
  → 检查围墙耐久、防御塔状态
  → 决定哪些英雄回防操控防御塔

【黑夜 — 80 回合】
  → 兽潮来袭（基础 + 敌方购买）
  → 英雄站在塔旁操控攻击
  → 留在野外的英雄被野兽攻击 → 极大概率阵亡
  → 建筑受损 → 生成修复清单
  → 黎明 → 阳光杀死残余兽群

【次日】
  → 循环往复，至第 10 天结束
```

### 13.3 终局结算

- 第 10 晚兽潮结束后触发终局结算
- 统计双方积分（含 Agent 任务积分）
- 积分上传全服大榜
- 生成本局数据报告

## 十五、设计亮点总结

1. **三角色分工**：开拓者（贸易/任务）、采集英雄（采集）、建造英雄（建造/维修/合成），各司其职
2. **阵营资源绑定 → 强制外交**：铜矿只有A能采、硅矿只有B能采，芯片必须通过中立商人交换获得
3. **渐进式合作**：Lv1→Lv2纯自给 → Lv2→Lv3自给（铁/硫磺双方均有）→ Lv3→Lv4必须合作（芯片=铜+硅）
4. **双轨经济 + 金币博弈**：以物易物强制合作 + 金币体系驱动情报套利和配额抢占
5. **塔需英雄操控 → 自然回防**：最多4座塔、3名英雄，需不断换位操控不同方向的塔
6. **竞争性合作（无PVP）**：不能攻击对方，比的是"谁的运营更高效""谁的任务完成得更快"，但可通过买野兽增加对方防守压力
7. **3类Agent任务全覆盖**：推理类（因果推理与博弈）、长上下文（跨天记忆整合）、自进化类（Web操作/文件操作/代码bugfix/数据库检索——工具使用与问题解决，子类型持续扩展）
8. **双通道任务架构**：任务官通道（自进化类，每局 1 种类型，主动接取）+ 世界新闻通道（推理类/长上下文，每日自动推送），职责清晰不混乱
9. **Agent 自进化设计**：每局随机 1 种自进化任务类型，同类型跨对局反复出现；类型内多道题目逐题递进（如 Web操作 T1→T2→T5）。Agent 通过总结该类型 SOP、优化策略，在后续对局中同类型任务上表现越来越好——跨对局的能力成长，而非一局内的灵活应变
10. **英雄可复活、开局仅基地**：鼓励大胆博弈，不怕失误；开局仅基地，布局完全自由
11. **日夜节奏清晰**：白天120回合探索/采集/任务，夜晚80回合集中回防，黎明阳光清场
