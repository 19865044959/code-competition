# Lux AI Season 3 — 大规模多智能体元学习对抗

> **类型**：学术竞赛 / MARL / 元学习
> **赛事**：NeurIPS 2024 官方竞赛（Kaggle 平台）
> **奖金**：$10,000（冠军）
> **关键词**：1v1对抗、16子Agent、元学习、部分可观测、JAX加速

---

## 一、比赛核心理念

**1v1 对抗**，但每个玩家同时控制 **16 个子 Agent**。环境是部分可观测的，且游戏机制在每局之间**动态变化**。

核心挑战：参赛 Agent 需要在**连续 5 局**中，前几局**探索**游戏机制（学习规则），后几局**利用**学到的知识击败对手。这是"元学习"的终极测试。

---

## 二、游戏设计

### 2.1 环境配置

| 参数 | 说明 |
|------|------|
| 地图 | 24×24 网格 |
| 子Agent数量 | 每方 16 个（共 32 个同时在场） |
| 可观测性 | 部分可观测（视野有限） |
| 资源 | 能量、矿石（分布在全局需要探索） |
| 游戏机制 | **每局动态变化**（如：资源生成率、移动成本、单位属性） |
| 单局时长 | ~10 分钟（1000 步） |
| 对局模式 | 连续 5 局 |

### 2.2 核心挑战

```
局 1: 探索 → 局 2: 适应 → 局 3: 优化 → 局 4: 对抗 → 局 5: 决胜
```

- **探索 vs 利用**：前几局花多少时间学习规则？
- **元策略**：如何在新规则下快速制定有效策略？
- **多子Agent协调**：16 个单位如何分工协作？
- **对手建模**：识别对手的学习速度并针对性压制？

---

## 三、技术亮点

### 3.1 JAX 高性能模拟
- 环境用 JAX 编写，**GPU 加速并行**
- 单 GPU 可同时跑数千局
- 训练成本大幅降低（相比传统 MARL 框架）

### 3.2 TrueSkill 排名系统
- 使用微软 TrueSkill（贝叶斯排名算法）
- 比 ELO 更精确的多方排名
- 已在 Kaggle 平台部署

### 3.3 获胜方案特点
- 多数使用 **Meta-RL**（如 RL²、VariBAD）
- 结合**规则基础**的快速适应模块
- Transformer 架构处理多子Agent关系

## 四、对"白日梦"赛制的启示

| Lux AI 设计 | 可借鉴点 |
|------------|---------|
| **多子Agent协调** | 白日梦4Agent分工可以更细粒度，如"侦查组2+执行组2" |
| **动态规则变化** | 可以设计"每轮副本物理规则不同"的机制 |
| **连续多局元学习** | 初赛20局的设计可以强调"局间学习"的考察 |
| **GPU加速评测** | 大规模穷举评测的技术方案 |

## 五、技术可行性

- **框架已开源**（Kaggle + JAX 环境）
- **GPU 需求**：单张 T4/A10 足够训练
- **适合 RL 赛道**选手（比 LLM 赛道更侧重算法）
- **排行榜自动化**：TrueSkill 方案成熟
