# SimWorld — 物理+社交世界模拟 (NeurIPS 2025 Spotlight)

> **类型**：学术平台 / LLM+VLM Agent 模拟器
> **来源**：NeurIPS 2025 Spotlight 论文
> **引擎**：Unreal Engine 5
> **关键词**：UE5、物理真实、社交动态、多模态、长时序任务

---

## 一、核心理念

构建在 **Unreal Engine 5** 上的高保真模拟器，用于评估 LLM/VLM Agent 在**物理真实+社交动态**的世界中的表现。支持多模态世界输入和开放词汇动作输出。

---

## 二、关键特性

| 特性 | 说明 |
|------|------|
| **引擎** | Unreal Engine 5（3A 级图形保真度） |
| **物理真实** | 碰撞、重力、光照、材质全部物理模拟 |
| **社交动态** | Agent 之间有社交关系、对话、协作/竞争 |
| **多模态输入** | 视觉（RGB+深度）、文本（对话/描述）、状态数据 |
| **开放动作空间** | Agent 可以用自然语言描述任何动作（不限于预定义动作集） |
| **长时序任务** | 多Agent协作外卖配送——需要协调与合作 |

---

## 三、已测试的前沿模型

| 模型 | 性能表现 |
|------|---------|
| Gemini | 在导航任务中表现最佳 |
| Claude | 在社交推理任务中表现最佳 |
| GPT-4o | 综合表现均衡 |
| DeepSeek | 在长时序规划任务中表现突出 |

### 关键发现
- 单Agent简单任务：所有模型表现良好
- **多Agent协调任务**：所有模型大幅下降（包括最强的 Gemini）
- 长时序+多Agent：**所有模型失败率 > 60%**——这是一个**开放研究问题**

## 四、与现有案例的对照

| SimWorld 特性 | 对你项目的启示 |
|--------------|--------------|
| UE5 引擎 | 是否需要图形界面？还是纯文本/API？|
| 多模态 | 你的 Agent 是否需要视觉输入？|
| 开放动作 | 是否允许 Agent 自由定义动作？|
| 长时序 | 30 天（荒野求生）vs 单局任务——时间尺度对比 |
| 社交维度 | Aivilization 的好友/事件系统类似 |

## 五、比赛设计考虑

SimWorld 揭示了一个重要现实：**当前最强模型在多Agent开放世界中的表现仍很差**。这意味着：

1. **比赛门槛要合理**：不能设计得太难，否则没有队伍能取得有意义的成绩
2. **分层难度设计**：初级关卡只考验单Agent能力，高级关卡才加入多Agent协调
3. **文本优先**：除非你的比赛受众有大量 CV 背景选手，否则纯文本API更友好
4. **SimWorld 本身可作评测平台**：开源，可作为"决赛可视化展示"的底座
