# Overcooked AI — 人机协作协调基准

> **类型**：学术基准 / 协作 MARL / 零样本协调
> **来源**：UC Berkeley Human Compatible AI 实验室 (NeurIPS 2019)
> **最新版本**：OvercookedV2 (2025)
> **关键词**：零样本协调、人机协作、分工博弈、部分可观测

---

## 一、核心理念

基于热门游戏《胡闹厨房》(Overcooked)，AI Agent 扮演厨师，必须与另一个 Agent（或真人）**紧密协作**完成上菜任务。核心考验的是**在没有预先约定的情况下，如何与陌生搭档高效协作**——即"零样本协调" (Zero-Shot Coordination)。

---

## 二、游戏机制

### 2.1 基础规则

```
厨房布局：网格地图
任务流程：
  取洋葱 → 切碎 → 放到锅里煮 → 装盘 → 上菜 → 得分

每个环节都需要：
- 动作协调（避免碰撞、堵塞通道）
- 策略协调（谁切菜？谁煮？谁上菜？）
```

### 2.2 关卡设计

| 布局名称 | 协作挑战 |
|----------|---------|
| Cramped Room | 狭窄空间，必须精确交替移动 |
| Coordination Ring | 环形布局，必须顺时针/逆时针配合 |
| Forced Coordination | 一方放食材，另一方必须从另一侧取 |
| Asymmetric Advantages | 双方拥有不对称的资源/位置优势 |
| 随机新布局 (OGC) | 测试从未见过的厨房布局的泛化能力 |

---

## 三、OvercookedV2 升级 (2025)

发现原版局限性后的重要升级：

| 升级点 | 说明 |
|--------|------|
| **不对称信息** | Agent 看到的信息不同 |
| **有意义的部分可观测** | 不是所有信息都对双方可见 |
| **随机性** | 环境动态变化 |
| **协议形成** | Agent 必须在测试时在线形成协作协议，不能依赖预训练约定 |

---

## 四、关键技术方案

### 4.1 主要方法流派

| 方法 | 描述 |
|------|------|
| **人口训练 (PBT)** | 训练多样化Agent群体→挑选最佳协作者 |
| **人感知训练 (HAT)** | 用人类行为模型训练Agent |
| **适应性策略 (AdaPT)** | Agent自带"适应模块"识别搭档风格 |
| **LLM-Agent协作** | 用LLM进行意图推理和沟通(Collab-Overcooked, EMNLP 2025) |

## 五、作为 Agent 比赛的改编设计

### 赛制建议：《星际厨房》— 人机协作烹饪大赛

```
世界观：一艘星际游轮上的厨房，服务员都是AI

角色设计（4 Agent 团队）：
- 主厨 Agent：负责烹饪策略与调度
- 切配 Agent：原材料加工
- 炉灶 Agent：火候控制、出锅时机
- 传菜 Agent：上菜路线规划、避撞

比赛模式：
A. 纯 Agent 协作模式
   - 你的 4 个 Agent 组成完整厨房团队
   - 与其他队伍的厨房比拼效率

B. 跨队混合协作模式
   - 随机打散队伍 → 不同队伍 Agent 混编
   - 考验：与陌生 Agent 的协作能力（零样本协调）

C. 人机协作模式
   - 一个人类玩家 + N 个 Agent
   - 考验：Agent 理解和配合人类的能力

评分：
- 上菜数量（效率）
- 菜品质量（0出错率）
- 协作流畅度（无碰撞、无死锁）
- 泛化能力（新厨房布局适应速度）
```

### 核心考验点
- **动作级协调**：精确的时序配合
- **分工策略**：动态任务分配
- **搭档适应**：快速理解搭档风格
- **死锁预防**：避免资源竞争导致的僵局

### 差异化优势
- **轻松有趣**：烹饪主题无暴力、老少皆宜
- **看得懂**：观众直觉理解"做菜效率"
- **协作本质**：纯粹的合作博弈
- **可扩展**：新食材、新厨房布局无限生成
