## 整体通信说明

每局比赛双方选手代码同步启动，选手程序以 HTTP Server 方式运行并监听指定端口；判题系统通过 HTTP 请求下发当前全量地图状态，接收选手返回的角色调度指令。

程序启动命令示例：

```Plain Text
bash run.sh port
```

- **port**：系统传入的监听端口号

- **服务监听地址**：固定为 0\.0\.0\.0

# 1 请求报文（Request）顶层结构

|字段|数据类型|字段说明|
|---|---|---|
|roundNo|Int|当前回合数|
|mapInfo|RequestMap|地图整体信息|
|teamOur|TeamOur|我方队伍全部信息|
|teamEnemy|TeamEnemy|敌方队伍可见信息|
|phaseTask|String|当前已领取任务原文描述|
|llmResp|String|LLM 返回响应内容|
|lastRoundActionResult|Map\<int, boolean\>|上回合各角色执行结果，key：角色ID；value：本次行动是否执行成功|
|ragQueryResult|String|RAG 检索返回结果|
|errors|Error\[\]|本轮产生的错误信息数组|

## 1\.1 RequestMap 地图信息结构

|字段|数据类型|字段说明|
|---|---|---|
|height|Int|地图 Y 轴总高度|
|width|Int|地图 X 轴总宽度|
|zones|Zone\[\]|地图固定中立障碍物数组|

### 1\.1\.1 Zone 中立区域元素

|字段|数据类型|字段说明|
|---|---|---|
|pos|Pos|坐标点位|
|neutralType|String|中立元素类型：mountain = 山地water = 水域|

**备注**：Zone 仅存放固定地形，不含空地、防御塔、基地、英雄、小兵、野怪。

#### 1\.1\.1\.1 Pos 坐标结构体

|字段|数据类型|字段说明|
|---|---|---|
|x|Int|X 坐标|
|y|Int|Y 坐标|

## 1\.2 TeamOur 我方队伍信息

|字段|数据类型|字段说明|
|---|---|---|
|type|String|阵营标识：challenger：挑战方defender：被挑战方|
|teamId|String|队伍唯一 ID|
|teamName|String|队伍名称|
|diamondNum|Int|当前剩余钻石数量|
|totalScore|Int|队伍累计总积分|
|playerTasks|PlayerTask\[\]|全部任务点信息列表|
|roles|Role\[\]|我方所有单位（塔 / 基地 / 小兵 / 英雄）列表|

### 1\.2\.1 Role 单位通用属性

|字段|数据类型|字段说明|
|---|---|---|
|id|Int|角色 / 单位唯一编号|
|pos|Pos|单位所在坐标；基地为 2\*2 占地，该坐标为基地左下角点位|
|roleType|String|单位分类|
|attackPower|Int|攻击力数值|
|attackRange|Int|攻击距离（切比雪夫距离）|
|skill1|Skill|一号技能信息|
|skill2|Skill|二号技能信息|
|health|Int|当前剩余血量|
|belong|String|所属线路：空字符串 = 无固定线路up = 上路mid = 中路down = 下路|

**roleType 取值明细**：

- **建筑物**：tower（防御塔）、station（基地）

- **小兵**：infantry（步兵）、cavalry（骑兵）、archer（弓箭兵）

- **英雄**：wallE（瓦力）、monkeyKing（大圣/孙悟空）、hellWarrior（地狱武士）、skeletonMage（骷髅法师）

### 1\.2\.2 PlayerTask 任务点信息

|字段|数据类型|字段说明|
|---|---|---|
|taskType|String|任务大类|
|coldDownRounds|int|任务刷新冷却剩余回合，0 = 任务点可领取；大于 0 = 还需等待对应回合解锁|
|taskPosition|Pos|任务点坐标|
|isValid|bool|当前任务点是否允许领取任务|
|diamondReward|int|完成任务可得钻石基数|
|scoreReward|int|完成任务可得积分基数|

### 1\.2\.3 Skill 技能结构

|字段|数据类型|字段说明|
|---|---|---|
|coldDownRounds|int|技能剩余冷却回合数|
|skillId|int|技能编号：1 = skill1，2 = skill2|

## 1\.3 TeamEnemy 敌方信息

|字段|数据类型|字段说明|
|---|---|---|
|roles|Role\[\]|视野内敌方所有单位信息|

## 1\.4 Error 错误结构体

|字段|数据类型|字段说明|
|---|---|---|
|errorCode|Int|错误编码：0：未知错误1：任务超时2：答案错误3：网络错误|
|description|String|错误文字描述|

# 2 响应报文（Response）顶层结构

|字段|数据类型|是否必选|说明|
|---|---|---|---|
|roleCommandMap|Map\<int, RoleCommand\>|M（必填）|全角色指令集合，key：角色 ID；value：对应角色动作指令|

## 2\.1 RoleCommand 单角色指令结构

|字段|数据类型|是否必选|说明|
|---|---|---|---|
|action|String|M（必填）|执行动作标识，可选值：nothing：无操作move：移动queryTaskInfo：查询任务信息queryLLM：调用 LLM 查询queryRag：调用 RAG 查询sqlInteraction：执行数据库 SQLsubmitAnswer：提交任务答案attack：普通攻击skill1：释放一号技能skill2：释放二号技能|
|targetPos|Pos|非必选|移动目标坐标 / 普攻目标坐标|
|skillTargetPos|Pos\[\]|非必选|指向型技能的目标坐标数组|
|taskAnswer|String|非必选|任务作答内容|
|queryInfo|String|非必选|对应 action 的入参数据：queryLLM → 传入 PromptqueryRag → RAG 请求报文sqlInteraction → 待执行 SQL 语句|
