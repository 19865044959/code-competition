# WhoisSpy — AI 社交推理锦标赛 · 设计文档

---

## 游戏概述

- **多智能体社交推理**：每局 4~6 支队伍同桌竞技。平民阵营拿到同一个词，卧底拿到相似但不同的词。通过轮番发言、交叉质询、投票淘汰，平民要找出卧底，卧底要伪装存活。
- **核心信息差**：卧底**只知道自己的词**，不知道平民词是什么——只能从平民发言中推测并模仿。这是与传统人类局（卧底通常能猜到平民词）最大的不同，也是 AI 比赛区分度的核心来源。
- **全赛程瑞士制**：不比单场输赢，比累计积分排名。每轮随机分组、随机角色，所有队伍打相同轮数，最终按总分截断晋级。
- **双方都必须使用 AI Agent**：Agent 需要理解词语语义、生成伪装/推理发言、分析他人发言中的矛盾、做出投票决策。传统 NLP 算法（关键词匹配、文本相似度）无法完成——因为比赛考察的是语义层面的推理、欺骗与识破。
- **核心机制一览**：
  - **三层词库 + LLM 动态校准**：L1（同范畴）/ L2（跨范畴）/ L3（抽象关联），每局开局前由 LLM 为当桌参赛模型量身生成词对，答案预置锁定，保证公平。
  - **自由发言 + 后台标签**：Agent 只输出发言文本和投票决策，策略标签（模糊化/具体化/转移焦点/质疑/附和/自证）由系统根据发言内容自动分类，仅用于观众端可视化，Agent 完全无感。
  - **纯客观积分**：不依赖 LLM 裁判，按存活轮次 + 投票正确率 + 嫁祸效果三维度计分。
  - **三层观众可视化**：上帝视角（身份+信任度+词对比）+ 发言流（实时文字）+ 推理回放（投票依据+揭晓），观众比选手知道更多，产生"上帝视角的焦虑感"。

## 一、为什么是"谁是卧底"

### 1.1 社交推理类游戏的对比

| 维度 | 谁是卧底 | 狼人杀 | 鹅鸭杀 |
|------|---------|--------|--------|
| 玩家数 | 4-8 人 | 6-12 人 | 4-15 人 |
| 身份种类 | 2 种 | 多种（狼/预言家/女巫/猎人/守卫） | 30+ 种 |
| 核心机制 | 词语推理 + 模糊发言 | 身份推理 + 技能博弈 | 任务完成 + 社交欺骗 |
| 信息量 | 极少（只有一个词不同） | 中等（角色技能提供信息） | 丰富（任务进度 + 行为观察） |
| Agent 难度 | ★★☆ | ★★★ | ★★★★ |
| 观众理解门槛 | 30 秒 | 5 分钟 | 15 分钟 |
| 每局时长 | 2-4 分钟 | 5-10 分钟 | 10-20 分钟 |
| 并行度 | 高（多桌同时） | 中 | 低 |

**选择"谁是卧底"的核心理由**：

1. **规则极简，观众 30 秒就能参与推理**。台下的每个观众都能自己判断"这句话听起来像平民词还是卧底词"。
2. **核心挑战恰好是 LLM 的最强能力**：语义区分、模糊表达、矛盾检测——不是比算力，是比语言理解和策略。传统算法完全无法参与。
3. **节奏紧凑**：一轮发言+投票约 90 秒，一局最多 4 轮（6 人桌）或 3 轮（4 人桌），适合大规模并行和直播节奏。
4. **区分度来源明确**：弱 Agent 秒暴露（发言太模糊/太具体），强 Agent 能以假乱真、精准识破。

### 1.2 与 Escape 的关系

本设计与 Escape（非对称对抗锦标赛）共享同一套设计哲学：

| 设计原则 | Escape | WhoisSpy |
|---------|--------|----------|
| AI 必要性 | 双层架构（语义+执行），传统算法无法破解像素谜题 | 语义推理+欺骗，传统 NLP 无法胜任 |
| 谜题生成 | LLM 开局批量生成，答案预置 | LLM 开局生成词对，答案预置 |
| 双 LLM 隔离 | 生成（高 temp）+ 评判（低 temp） | 生成（高 temp）+ 违规检测（低 temp） |
| 全程可审计 | 像素日志+逐帧回放 | 发言+投票+推理全量日志 |
| 纯客观积分 | 多维度计分，无 LLM 裁判 | 存活轮次+投票正确率，无 LLM 裁判 |
| 策略标签 | — | 系统自动分类，Agent 无感，仅观众可见 |

## 二、词语系统

### 2.0 设计理念

谁是卧底的灵魂在于词对 —— 两个词太像则卧底容易蒙混，太不像则卧底秒暴露。找到一个"恰到好处"的词对，是整个比赛质量的根基。

#### 人类的词对 vs AI 的词对

```
人类局：主持人手动选词
  - 词对来源：日常生活经验
  - 难点：选手可能恰好不知道某个词
  - 策略：选"大家都知道但容易混淆"的词对

AI 局：LLM 动态生成
  - 词对来源：LLM 的语义空间
  - 难点：AI 什么词都知道 → 信息差不在"知识"而在"推理"
  - 策略：选"共享足够多属性但恰好有可被观察到的差异"的词对
```

### 2.1 三层词库结构

```
L1: 同范畴词对（基础难度）
  - 共享同一个上位词
  - 属性高度重叠，但有 1-2 个关键差异
  - 示例：枕头 vs 靠垫 | 碗 vs 碟 | 雨衣 vs 风衣
  - 使用场景：海选赛早期轮次，确保所有队伍能参与

L2: 跨范畴词对（中等难度）
  - 不同类别，但共享 3+ 个可观察属性
  - 需要精细语义区分
  - 示例：闹钟 vs 手表 | 电梯 vs 扶梯 | 字典 vs 百科全书
  - 使用场景：海选赛后期 + 晋级赛，拉开排名差距

L3: 抽象关联词对（高难度）
  - 需要深层语义推理
  - 共享抽象属性（情感维度、社会功能等）
  - 示例：嫉妒 vs 羡慕 | 习惯 vs 瘾 | 秘密 vs 谎言
  - 使用场景：半决赛 + 总决赛，决定冠亚军归属
```

### 2.2 LLM 动态校准

```
开赛前系统调用 LLM（temperature=1.0）：

Prompt 结构：
  你正在为一场 AI 谁是卧底比赛生成词对。
  
  本桌参赛模型：[列出该桌 6 个模型名称]
  比赛阶段：[海选/晋级/半决赛/决赛]
  难度要求：[L1/L2/L3]
  
  请生成 3 组词对，每组满足：
  - 两个词有 3-5 个可被观察到的共同属性（如用途、场景、外观）
  - 有 2-3 个可被推理发现的关键差异
  - 两个词都是常见事物/概念，不涉及极端专业知识
  - 卧底用模糊化策略可在前 2 轮不被发现的概率约 40-60%
  - 禁止使用：同义词、方言词、已在本赛事出现过的词对
  
  返回格式：
  {
    "pair_1": {
      "civilian_word": "...",
      "spy_word": "...",
      "shared_attributes": ["...", "..."],   // 内部使用，不暴露
      "key_differences": ["...", "..."]       // 内部使用，不暴露
    },
    "pair_2": {...},
    "pair_3": {...}
  }

→ 系统预存 3 组，比赛时随机抽 1 组
→ 每桌独立生成，各桌词对不同
→ 全局去重：已用词对永不重复
```

### 2.3 词对质量保证

| 验证项 | 方法 | 不通过时 |
|--------|------|---------|
| 两个词不完全相同 | 字符串比对 | 重新生成 |
| 两个词不能是明显的同义词 | LLM 复审（temperature=0.1） | 重新生成 |
| 两个词都是常见词汇 | 词频数据库检查 | 标记、人工审核 |
| 词对未被使用过 | 全局词库去重检查 | 重新生成 |
| 两个词之间不存在"谁包含谁" | 语义包含关系检查 | 重新生成 |

## 三、回合制流程

### 3.0 设计理念

传统"谁是卧底"是自由发言 → 投票 → 淘汰。AI 比赛在此基础上做了一个关键创新：

**Agent 只管说人话，标签由系统自动归类**。Agent 的输出只有两样东西：发言文本 + 投票决策。策略标签（模糊化/具体化/转移焦点/质疑/附和/自证）由独立分类器在发言提交后实时分析生成，Agent 完全看不到标签，Agent 之间也只能看到实际发言内容。

这样做的好处：
- Agent 专注在"说什么"和"投谁"，不需要额外学习"选标签"这套元语言
- 标签不受 Agent 自身策略干扰（避免 Agent 选"自证"标签但发言内容其实在"转移焦点"这种欺骗）
- 观众看到的是系统客观分析的结果，比 Agent 自报标签更可信

### 3.1 单轮流程

```
┌─────────────────────────────────────────────────┐
│  阶段 1：发言（15 秒）                            │
│  Agent 收到当前游戏状态                           │
│  → 生成一句话发言                                 │
│  → 系统后台自动分类策略标签（Agent 无感）           │
├─────────────────────────────────────────────────┤
│  阶段 2：公开辩论（20 秒）                         │
│  所有发言同时公开（纯文本，不含标签）               │
│  Agent 分析他人发言：                             │
│    - 谁的描述和我的词有偏差？                      │
│    - 谁在回避具体描述？谁在带节奏？                 │
│    - 谁的行为模式和上一轮不一致？                   │
│  → 更新内部嫌疑评估                               │
├─────────────────────────────────────────────────┤
│  阶段 3：投票（10 秒）                            │
│  每个 Agent 投出一票 + 一句话推理依据               │
│  → 得票最高者被淘汰 → 公布身份                     │
│  → 检查胜负条件                                   │
└─────────────────────────────────────────────────┘

单轮总计：~90 秒（含结算过渡约 3 分钟）
```

### 3.2 策略标签（系统自动分类，仅观众可见）

策略标签由独立 LLM 分类器（temperature=0.1）在 Agent 发言后实时分析生成。**Agent 完全看不到标签**——Agent 之间只能看到实际发言文本。标签仅服务于观众可视化和赛后分析。

| 标签 | 典型发言特征 | 观众看到的信号 |
|------|------------|--------------|
| **模糊化** | 概括、笼统、避免具体细节 | "这个人在兜圈子" |
| **具体化** | 包含可验证的具体属性 | "这个人在给出线索" |
| **转移焦点** | 主要讨论他人而非描述自己 | "这个人在回避描述" |
| **质疑他人** | 直接指出他人的矛盾 | "这个人发起进攻" |
| **附和他人** | 赞同或强化他人的描述 | "这个人在站队" |
| **自证** | 通过更多细节证明自己 | "这个人在洗清嫌疑" |

### 3.3 策略标签的区分度（仅对观众有意义）

由于标签是系统自动分类而非 Agent 主动选择，标签的区分度体现在**观众能从标签流中看出什么**：

- **弱 Agent**：发言内容空洞、自相矛盾，标签频繁切换（系统也判断不准它在干什么）
- **中等 Agent**：发言风格单一，卧底一直模糊化、平民一直具体化——可被对手识破
- **强 Agent**：卧底会在模糊化和刻意具体化之间动态切换（反套路）；平民先附和收集信息再精确质疑——标签流呈现出"有策略的节奏变化"

标签切换频率、标签与发言内容的一致性、标签与投票行为的一致性 —— 都是观众和赛后分析的关键看点。

## 四、积分体系

### 4.0 设计理念

核心约束：
1. **纯客观** —— 不接受 LLM 裁判，所有积分由可验证的客观事件决定
2. **角色分离** —— 卧底分和平民分独立计算，不因"运气好多当了平民"而占优
3. **精细区分** —— 不只分输赢，同是"平民赢"也有 60/80/100 分的差距
4. **无惩罚性扣分** —— 错误的投票不扣分（只不得分），鼓励大胆决策

### 4.1 卧底积分

```
卧底积分 = 存活奖励 + 嫁祸加分

存活奖励（取最高，不累加）：
  存活到最后（只剩 2 人）    100 分
  在第 3 轮被投出（4人桌）
  或第 4 轮被投出（6人桌）    60 分
  在第 2 轮被投出             30 分
  在第 1 轮被投出              0 分

嫁祸加分：
  每成功投票淘汰 1 个平民    +10 分
  （即：你投了平民，且该平民当轮被淘汰）
```

**设计意图**：卧底的目标不是"赢"，而是"活得久+制造混乱"。一个存活 3 轮且成功嫁祸了 2 个平民的卧底（80 分），比一个存活到最后但全程沉默的卧底（100 分）更值得高分。

### 4.2 平民积分

```
平民积分 = 胜利奖励 + 投票正确加分

胜利奖励（取最高，不累加）：
  在第 1 轮正确投出卧底       100 分
  在第 2 轮正确投出卧底        80 分
  在第 3+ 轮正确投出卧底       60 分
  卧底获胜                     0 分

投票正确加分：
  每轮投票正确（投了卧底）    +10 分
  每轮投票错误（投了平民）      0 分
```

**设计意图**：
- 最快识破卧底的平民获得最高荣誉（100 分）—— 鼓励强推理
- 持续正确判断的平民即使不是最终投出卧底的人，也能通过累计 +10 分获得加分
- 投错不扣分 —— 避免"不敢投"的保守策略

### 4.3 总分与排名

```
总积分 = Σ 每局得分

每局得分：
  - 当卧底时 → 按卧底积分公式计算
  - 当平民时 → 按平民积分公式计算

同分裁决顺序：
  1. 正确投票率 = 投票正确的轮次 / 总投票轮次
  2. 卧底平均存活轮次
  3. 平票后补投正确率
```

## 五、赛程与瑞士制

### 5.0 设计理念

**为什么不能用淘汰制？**

1v1 比赛中，胜者晋级败者淘汰 —— 简单直接。但多人桌（6 人）无法定义单一的"胜者"：如果有人投对了卧底、有人投错了，谁是"胜者"？如果卧底是 Team A，Team B 第二轮识破了，Team C 第三轮才识破 —— 谁更强？

因此采用**全赛程瑞士制**：所有队伍打相同轮数，每轮重新分组，最终按累计积分排名截断。

### 5.1 赛程结构

```
┌─────────────────────────────────────────────────────┐
│  阶段        队伍    轮数   桌型       并行桌   耗时   │
├─────────────────────────────────────────────────────┤
│  海选赛      1000    8轮    6人桌      167桌   ~24min│
│  (截断前64)         (1卧底+5平民)                    │
│                                                      │
│  晋级赛       64     5轮    6人桌       11桌   ~15min│
│  (截断前16)         (1卧底+5平民)                    │
│                                                      │
│  半决赛       16     4轮    4人桌        4桌   ~12min│
│  (截断前4)          (1卧底+3平民)                    │
│                                                      │
│  总决赛        4     5轮    4人桌        1桌   ~15min│
│  (产生冠军)         (1卧底+3平民)                    │
├─────────────────────────────────────────────────────┤
│  总计：~70 分钟比赛 + 过渡解说 ≈ 90 分钟直播          │
└─────────────────────────────────────────────────────┘
```

### 5.2 瑞士制配对算法

```
Round 1：
  所有队伍随机打乱 → 按顺序分组（每 6 人一组）

Round N (N ≥ 2)：
  1. 按当前总积分降序排列
  2. 从最高分开始，取前 6 人（或 4 人）作为第 1 桌
  3. 继续取下一组，直到所有队伍分配完毕
  4. 约束检查：
     - 如果某两个队伍已经同桌过 2 次 → 与相邻组交换队员
     - 如果无法满足约束 → 放宽约束（允许 3 次同桌）
  5. 每桌随机抽取 1 个卧底
     - 系统后台追踪每个队伍的卧底次数
     - 如果某队已连续 N 轮未当卧底 → 提高被抽中概率
     - 保证海选赛 8 轮中每队卧底 1-2 次
```

### 5.3 角色均衡保证

```
后台追踪表（以海选赛 8 轮为例）：

队伍    已当卧底次数    已当平民次数
Team_A      1              7
Team_B      2              6
Team_C      1              7
...

均衡约束：
- 第 1-6 轮：纯随机抽取卧底
- 第 7 轮起：如果某队卧底次数 = 0 → 强制安排为卧底
- 如果某队卧底次数 = 2 → 不再安排为卧底
- 最终结果：每队卧底 1-2 次，偏差 ≤1 次
```

### 5.4 为什么 16 进 4 改用 4 人桌

```
16 支队伍 ÷ 6 人桌 → 除不尽（16÷6=2余4）
选项 A：6+6+4（最后一桌 4 人） → 不对称
选项 B：全部降为 4 人桌 → 4×4=16，刚好分 4 桌 ✓

4 人桌（1卧底+3平民）的特点：
- 卧底存活轮次更少（最多 3 轮 vs 6人桌最多 4 轮）
- 每人的发言权重更高
- 投票容错率更低（1次错误投票可能导致平民失败）
- 更适合高水平对抗（半决赛和决赛）
```

## 六、观众可视化系统

### 6.0 设计理念

社交推理比赛的核心观赏性问题：如果观众不知道卧底是谁、不知道两个词是什么，他们看到的只是一群人在说一些模糊的话，然后投票 —— 毫无观赏性可言。

解决方案：**观众比选手知道更多**。观众同时看到平民词和卧底词，知道每个人的身份。这产生了"上帝视角"的独特体验 —— 观众可以自己判断每句话"像平民词还是卧底词"，产生"为什么他们还没发现？！"的参与感和焦虑感。

### 6.1 三层信息架构

```
第 1 层：上帝视角面板（比赛大屏，始终显示）
┌─────────────────────────────────────────────────┐
│  Round 3 │ 桌号: Table_07 │ 卧底仍存活 ⚠️        │
│  平民词: 保温杯    卧底词: 茶杯                   │
│                                                 │
│  ┌──────┬──────┬──────┬──────┬──────┬──────┐    │
│  │Team1 │Team2 │Team3 │Team4 │Team5 │Team6 │    │
│  │ 🟢   │ 🟢   │ 🔴   │ 🟢   │ 🟢   │ 💀   │    │
│  │████  │███░  │█░░░  │████  │███░  │ R2   │    │
│  │模糊化│具体化│转移  │附和  │质疑  │      │    │
│  └──────┴──────┴──────┴──────┴──────┴──────┘    │
└─────────────────────────────────────────────────┘

第 2 层：发言流（侧屏/手机第二屏）
  实时显示当前轮所有 Agent 的发言文本 + 系统自动归类的策略标签
  历史发言可滚动查看

第 3 层：推理回放（投票揭晓时全屏展示）
  被淘汰者的身份揭晓动画
  关键投票的推理依据展示
  错误投票的"错在哪"标记
```

### 6.2 信任度柱状图

信任度由后台引擎实时计算，仅观众可见：

```
信任度 = 100（基础值）
       - 15 × 被投票次数
       + 10 × 投票正确次数（投了卧底）
       - 10 × 被已淘汰者投票次数（"死人指认"负面信号）
       - 5  × 策略标签与历史不一致次数

更新频率：每轮投票结束后重新计算
显示方式：柱状图（满格 = 5 格），红色闪烁表示信任度骤降
```

### 6.3 解说辅助

```
解说员专用面板（额外信息）：
  - 每个 Agent 的策略标签切换历史
  - "关键转折点"标记（如：某人第一次被质疑、信任度骤降时刻）
  - 词对差异雷达图（两个词的属性相似度可视化）
  - 高光时刻 AI 自动剪辑标注
```

### 7.2 双 LLM 隔离

沿用 Escape 的设计原则：

| LLM 实例 | 用途 | temperature | 说明 |
|---------|------|------------|------|
| LLM-1 | 词语生成 | 1.0 | 高创造性，确保词对各不相同 |
| LLM-2 | 违规检测 | 0.1 | 确定性判断，检测发言是否直接暴露了词语 |
| LLM-3 | 策略分类 | 0.1 | 确定性归类，分析发言文本属于哪种策略模式（仅观众可见） |

### 7.3 并发架构

```
海选赛并发模型：
  167 桌同时开打
  → 每桌一个独立的 GameEngine 实例
  → 共享词语生成器（加锁，避免词对重复）
  → 共享积分榜（原子更新）

Agent 通信：
  → REST/gRPC 接口
  → 每回合推送状态，等待返回
  → 超时 = 该 Agent 单回合降级处理

可视化：
  → WebSocket 推送实时状态到观众端
  → 支持多桌同时观看（观众可切换桌）
  → 支持赛后回放（全量日志重放）
```

## 八、Agent 技术考察重点

| 考察维度 | 具体说明 |
|---------|---------|
| **语义区分能力** | Agent 必须理解自己拿到的词的核心语义，能够区分"哪些属性是这个词独有的"和"哪些属性可能和卧底词共享" |
| **伪装策略** | 卧底 Agent 需要推断平民词的大致范畴，生成既不完全暴露自己、又不至于过于模糊的发言——在"太具体（暴露差异）"和"太模糊（暴露身份）"之间找到平衡 |
| **矛盾检测** | 平民 Agent 需要从多轮发言中找出逻辑不一致的线索——谁的描述和其他人不同？谁在回避具体细节？谁的行为模式在关键轮次突变？ |
| **社会推理** | Agent 需要分析投票模式——谁在投谁？被淘汰者的投票是否有信息量？哪些人形成了"投票同盟"？ |
| **多步推理** | 单轮发言不足以判断 → 需要追踪多轮的历史发言和投票 → 构建嫌疑人的"行为画像" |
| **鲁棒性** | 自己被认为可疑时如何自证？队友被冤枉时如何辩护？信息不足时如何避免随机投票？ |
| **策略多样性** | 高级 Agent 能展现出丰富的策略层次：卧底刻意具体化来"反套路"；平民先附和再质疑来收集信息；卧底嫁祸后迅速转移焦点 |

## 九、与传统 NLP 算法的不可替代性

| 任务 | 传统算法能做什么 | 传统算法不能做什么 |
|------|---------------|------------------|
| 判断两个词的相似度 | Word2Vec/余弦相似度 | 不理解哪些属性是"人会在描述中提到的" vs "人不会注意到的" |
| 检测发言中的矛盾 | 文本相似度比对 | 不理解语义层面的矛盾（"可以用来装热水" vs "一般装冷水喝"） |
| 生成模糊发言 | 模板填充 / 同义词替换 | 无法在"太具体暴露词"和"太模糊暴露身份"之间做策略性平衡 |
| 投票决策 | 关键词匹配 + 规则 | 无法综合多轮发言历史、投票模式做贝叶斯推理 |
| 伪装推理 | — | 卧底需要从平民发言中推测平民词——这是典型的 Theory of Mind 推理，传统算法完全无法处理 |

## 十、异常与安全

### 10.1 异常处理

| 异常 | 处理 |
|------|------|
| Agent 超时（单次） | 发言：生成默认模糊发言；投票：随机投给存活者之一 |
| Agent 连续 3 次超时 | 视为弃权，自动淘汰，该队后续轮次不参与 |
| 发言直接包含词语 | LLM-2 实时检测 → 发言内容替换为模糊占位符 → 标记 VIOLATION |
| 发言包含攻击性内容 | 替换为 "[内容违规]" → 标记 VIOLATION |
| 连续 3 次 VIOLATION | 自动淘汰 |
| Agent 连接断开 | 同超时；连续 3 轮断开 → 弃权 |
| 词语生成失败 | 使用预置词库（50 组人工审核词对） |
| 某桌全票投同一人 | 标记异常进入审计队列，结果仍有效 |
| 并发冲突 | 每桌独立 GameEngine 实例，天然隔离 |

### 10.2 安全措施

| 措施 | 说明 |
|------|------|
| 词库全局去重 | 已使用词对永不重复，防止"背题库" |
| 全量日志 | 所有发言、策略、投票、推理全过程记录，赛后完整审计 |
| Agent 隔离 | Agent 之间无法互相通信，只能通过比赛引擎的公开信息交互 |
| 确定性回放 | 所有随机种子记录，可精确复现任何一局 |
| 作弊检测 | 不同队伍提交相同 Agent 代码 → 标记 → 人工审核 → 取消资格 |

## 十一、Loop 大循环架构

每局比赛的核心执行流程：

```
比赛开局 → LLM-1 为该桌生成 3 组词对 → 随机抽取 1 组
         → 分配角色（随机选 1 个卧底）→ 分发给各 Agent

while 比赛未结束:
    1. 发言阶段（15 秒）
       引擎推送状态 → Agent 生成发言文本 → 提交
       LLM-2 违规检测（并行）
       LLM-3 策略分类（并行，Agent 无感）
    
    2. 公开辩论阶段（20 秒）
       所有发言（纯文本）同时公开
       Agent 收到，内部分析（无需提交）
    
    3. 投票阶段（10 秒）
       Agent 提交投票目标 + 推理依据 → 提交
    
    4. 结算
       计票 → 淘汰最高票者 → 公布身份
       → 更新信任度（仅引擎内部+观众端）
       → 检查胜负条件
       → 记录日志

比赛结束 → 积分结算 → 更新积分榜 → 进入瑞士制下一轮配对
```
